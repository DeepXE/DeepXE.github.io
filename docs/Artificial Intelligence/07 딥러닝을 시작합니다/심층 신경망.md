### 렐루 활성화 함수
 - 초창기 인공 신경망의 은닉층에 많이 사용된 활성화 함수는 **시그모이드 함수**였다. 그러나 이 함수는 오른쪽과 왼쪽 끝으로 갈수록 그래프가 누워있기 때문에 올바른 출력을 만드는데 신속한 대응이 어렵다. 이는 특히 층이 많은 심층 신경망일수록 더욱 심해진다.
 - 위와 같은 단점을 개선하기 위해 제안된 활성화 함수가 렐루 함수이다.
   - 입력이 양수일 경우 입력을 통과시키고 음수일 경우에는 0으로 만든다.
   - 심층 신경망, 이미지 처리에서 뛰어난 모습을 보인다.
   
#### 표현
 - max(0, z)와 같이 쓸 수 있는데, 이 표현의 의미는 z가 0보다 크면 z를 출력하고 z가 0보다 작으면 0을 출력한다는 것이다.

### 옵티마이저

 - 케라스에서 지원하는 다양한 종류의 경사 하강법 알고리즘을 의미한다.


```python
# 가장 기본적인 확률적 경사 하강법인 SGD
model.compile(optimizer='sgd', loss='sparse_categorical_crossentropy', metrics='accuracy')
```


```python
# 위의 코드와 같은 내용
sgd = keras.optimizers.SGD()
model.compile(optimizer=sgd, loss='sparse_categorical_crossentropy', metrics='accuracy')
```


```python
# SGD 클래스의 학습률 변경 (기본값은 0.01)
sgd = keras.optimizers.SGD(learning_rate=0.1)
```


```python
# 모멘텀 최적화 : 매개변수 momentum을 0보다 큰 값으로 지정하면 이전의 그레이디언트를 가속도처럼 사용할 수 있다. 일반적으로 매개변수는 0.9이상을 지정한다.
# SGD 클래스의 nesterov 매개변수를 True로 바꾸면 네스테로프 모멘텀 최적화(=네스테로프 가속 경사)를 사용할 수 있다. (기본값은 False)
# 네스테로프 모멘텀 최적화 : 모멘텀 최적화를 2번 반복하여 구현한다. 일반적으로 기본 확률적 경사 하강법보다 더 나은 성능을 제공.

 - 학습률 매개변수를 튜닝하는 수고르 덜 수 있는 것이 장점.
 - 대표적인 옵티마이저로 Adagrad와 RMSprop이 있다.

### 🌱 정리

> 심층 신경망 : 2개 이상의 층을 포함한 신경망. 종종 다층 인공 신경망, 심층 신경망, 딥러닝을 같은 의미로 사용.
> 렐루 함수 : 이미지 분류 모델의 은닉층에 많이 사용하는 활성화 함수. 시그모이드 함수는 층이 많을수록 활성화 함수의 양쪽 끝에서 변화가 작기 때문에 학습이 어려워짐. 그러나 렐루 함수는 이런 문제가 없으며 계산도 간단함.
> 옵티마이저 : 신경망의 가중치와 절편을 학습하기 위한 알고리즘 또는 방법. 케라스에는 다양한 경사 하강법 알고리즘이 구현되어 있는데 대표적으로 SGD, 네스테로프 모멘텀, RMSprop, Adam 등이 있음.

dense1=keras.layers.Dense(100, activation='sigmoid', input_shape=(784, ))

dense2=keras.layers.Dense(10, activation='softmax')
```
:::info 은닉층과 출력층의 활성화 함수
* 활성화 함수: 신경망 층의 선형 방정식 계산 값에 적용하는 함수
* 출력층 활성화 함수
    * 이진 분류: 시그모이드 함수
    * 다중 분류: 소프트맥스 함수
* 은닉층 활성화 함수: 비교적 자유로움
:::

:::danger 은닉층의 활성화 함수의 적용 이유
* 은닉층에 활성화 함수를 적용하는 이유
    * 은닉층에서 선형적인 산술 계산만 수행하면 이전의 선형 방정식에서 나온 결과를 그대로 사용하기 때문에 수행 역할(의미)가 없어짐
    > a * 4 + 2 = b, b * 3 - 5 = c -> a * 12 + 1 = c
    >
    > b의 역할이 없어진다.
    * 그러므로 은닉층의 결과에 함수를 취하여 역할을 할 수 있게 한다.
    > log(b)=k -> k * 3 - 5 = c
    >
    > log(b)를 k로 치환하여 단순히 합쳐지는 연산이 아니게 됨

* 은닉층의 뉴런 개수: 특별한 기준은 없으나 출력층의 뉴런보다는 많아야 한다.
:::
