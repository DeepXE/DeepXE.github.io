5 검증 폴드의 시간, 모델 평가 확인


```python
#교차 검증 함수
from sklearn.model_selection import cross_validate

#fit_time, score_time, test_score를 key로 가지는 딕셔너리 반환
scores=cross_validate(dt, train_input, train_target)
print(scores)
```

    {'fit_time': array([0.00915456, 0.00731206, 0.00769567, 0.00747299, 0.00706005]), 'score_time': array([0.00099564, 0.00081968, 0.00077152, 0.00071406, 0.00077248]), 'test_score': array([0.86923077, 0.84615385, 0.87680462, 0.84889317, 0.83541867])}
    

:::info cross_validate 함수가 반환하는 딕셔너리 정보
* 각 key에 대한 value가 배열로 저장되어 있음
1. fit_time: 모델 훈련에 소요되는 시간(5폴드 - 5개)
2. score_time: 모델 검증에 소요되는 시간(5폴드)
3. test_score: 검증 폴드 점수(5개)

* 훈련 세트만 넘겨주면 알아서 검증 세트를 떼어 교차 검증을 시행함
* 훈련 세트를 몇 부분으로 나누냐에 따라 n 폴드 교차 검증이라고 부름(=n겹 교차 검증)
    * 3 폴드 교차 검증: 훈련 세트를 세 부분으로 나누어 교차 검증을 수행함.
:::

<br/>

5개의 검증 폴드 점수의 평균 출력


```python
import numpy as np

#위의 scores딕셔너리에서 test_score를 key로 가지는 value(검증 점수)의 평균 출력
print(np.mean(scores['test_score']))
```

    0.855300214703487

훈련 세트를 섞어주는 StratifiedKFold 분할기(default)


```python
#훈련 세트를 섞어주는 분할기
from sklearn.model_selection import StratifiedKFold

#cv의 default로 StratifiedKFold가 사용되어 위와 결과가 동일함
# -> 교차 검증 함수는 내부적으로 훈련 데이터를 섞어 사용함
scores=cross_validate(dt, train_input, train_target, cv=StratifiedKFold())
print(np.mean(scores['test_score']))
```

    0.855300214703487
    

:::info cross_validate()함수
* KFold 분할기: 회귀 모델
* StratifiedKFold: 분류 모델

* 위의 클래스들은 default이므로 따로 명시 안해도 된다
:::

<br/>

하이퍼파라미터 튜닝 - 데이터 설정(GridSearchCV클래스)


```python
#그리드 서치 클래스
from sklearn.model_selection import GridSearchCV

#하이퍼파라미터 탐색과 교차 검증을 한 번에 수행해줌 -> 앞의 cross_validate 함수 필요 없음
#params: dictionary(0.0001~0.0005까지 5개의 값 중 최적의 하이퍼파라미터를 찾음)
params={'min_impurity_decrease':[0.0001, 0.0002, 0.0003, 0.0004, 0.0005]}
```
<br/>

그리드 서치 클래스의 사용


```python
#5개의 값에 대해 총 5번 실행
#GridSearchCV에서 cv의 기본값은 5이므로 5 폴드 교차 검증 -> 총 25번의 시행
#n_jobs: cpu코어 수를 지정함
gs=GridSearchCV(DecisionTreeClassifier(random_state=42), params, n_jobs=-1)
```

:::info 그리드 서치 클래스
* 최적의 하이퍼파라미터 탐색과 교차 검증을 한꺼번에 하기 때문에, 앞의 cross_validate 함수를 사용하지 않아도 된다.
* 매개변수
1. 분류 클래스(회귀 클래스): 사용할 알고리즘 클래스 넣기(위의 예에서는 결정 트리)
2. params: 위의 알고리즘 클래스에서 하이퍼파라미터 튜닝을 진행할 딕셔너리 대입 
:::

<br/>

최적의 하이퍼파라미터, 폴드의 점수 출력


```python
gs.fit(train_input, train_target)
#25개의 모델 중 가장 점수가 높은 모델 -> 일반 결정 트리처럼 사용 가능
dt=gs.best_estimator_
print(dt.score(train_input, train_target))
```

* 해당 알고리즘 객체로 반환하여, 일반적인 클래스처럼 사용 가능

이전까지는 문제를 간단히 하려고 테스트 세트를 사용했지만, 테스트 세트로 일반화 성능을 올바르게 예측하려면 테스트 세트를 최대한 사용하지 말아야 함. > 마지막에 딱 한번 사용하는 것이 좋음.

테스트 세트를 사용하지 않으면 모델의 적합도를 판단하기 어려움. > 이를 해결하기 위해 나온 방법이 훈련 세트를 또 나누는 것. (=검증 세트)

import pandas as pd

data = wine[['alcohol', 'sugar', 'pH']].to_numpy()

from sklearn.model_selection import train_test_split

## 교차 검증

- 일반적으로 많은 데이터를 훈련에 사용할수록 좋은 모델이 만들어짐.

- 하지만 검증 세트를 너무 조금 떼어 놓으면 검증 점수가 불규칙적이고 불안정함.

- 검증 세트를 떼어 내어 평가하는 과정을 여러 번 반복(교차 검증)하면 안정적인 검증 점수를 얻고 훈련에 더 많은 데이터를 사용할 수 있음


```python
from sklearn.model_selection import cross_validate

scores = cross_validate(dt, train_input, train_target)
print(scores)
```

    > {'fit_time': array([0.00598311, 0.00598454, 0.00598383, 0.00598454, 0.00598192]), 'score_time': array([0.        , 0.        , 0.00099683, 0.00099826, 0.00099874]), 'test_score': array([0.86923077, 0.84615385, 0.87680462, 0.84889317, 0.83541867])}
    

fit_time :  모델을 훈련하는 시간

score_time : 검증하는 시간

각 키마다 5개의 숫자가 담겨 있음.


```python
import numpy as np

print(np.mean(scores['test_score']))        # 입력한 모델에서 얻을 수 있는 최상의 검증 점수 예측
```

    > 0.855300214703487
    

cross_validate()는 훈련 세트를 섞어 폴드를 나누지 않음. (앞서 train_test_split()으로 전체 데이터를 섞은 뒤 훈련 세트를 준비했기에 불필요했음)

그렇기에 **분할기**를 사용해 훈련 세트를 섞어줘야 함.


```python
from sklearn.model_selection import StratifiedKFold

scores = cross_validate(dt, train_input, train_target, cv=StratifiedKFold())
print(np.mean(scores['test_score']))
```

    > 0.855300214703487
    


```python
splitter = StratifiedKFold(n_splits=10, shuffle=True, random_state=42)      # n_splits = n (n-폴드 교차 검증)
scores = cross_validate(dt, train_input, train_target, cv=splitter)
print(np.mean(scores['test_score']))
```

    > 0.8574181117533719
