# 05-3 트리의 앙상블

## 정형 데이터와 비정형 데이터

* 정형 데이터: 어떠한 구조로 구성되어 있어 csv, 데이터베이스, 엑셀 등에 저장하기 용이한 데이터(지금까지 다룬 데이터)

* 비정형 데이터: 정형 데이터와 반대되는 개념, 식별 가능한 구조가 없는 형태 - 단순 데이터로는 별 의미가 없기 때문에 디코딩 등의 처리를 하여 유의미한 데이터로 변환 필요

## 랜덤 포레스트

* 랜덤 포레스트: 결정 트리를 랜덤하게 만들어, 여러 결정 트리의 예측을 사용하여 최종 예측을 만든다.
* 특징
    * 부트스트랩 샘플: 훈련 데이터에서 랜덤하게 데이터를 뽑은 훈련 데이터(중복추출 가능)

    * 전체 특성 중 일부 특성을 무작위로 고른 후 이 중에서 최선의 분할을 찾는다.
    
    * 기본적으로 100개의 결정 트리 훈련

    * RandomForestClassifier(분류): 전체 특성 수의 제곱근만큼의 특성 선택
        * 가장 높은 확률을 가진 클래스 선택
    * RandomForestResressor(회귀): 전체 특성 사용
        * 단순 예측의 평균 선택

    * 랜덤 샘플, 특성 덕분에 과대적합을 막아줌

## 엑스트라 트리

* 랜덤 포레스트와 매우 유사, 부트스트랩 샘플을 사용하지 않는다는 차이(전체 훈련 세트를 사용)

* 노드 분할 시 무작위로 분할 - splitter 매개변수가 random인 결정 트리들을 사용
(가장 좋은 분할을 찾는 랜덤 포레스트와 차이)

* 하나의 트리에서 특성을 무작위 분할한다면 성능이 낮아지지만, 트리가 많기 때문에 과대적합을 막는 효과가 있음

* 보통 엑스트라 트리의 무작위성이 더 크기 때문에, 더 많은 결정트리를 훈련하게 된다.

* 노드의 분할이 랜덤하기 때문에, 계산 속도가 랜덤 포레스트에 비해 더 빠르다.

## 그레이디언트 부스팅

* 깊이가 3인 결정 트리를 사용

* 깊이가 얕은 결정 트리를 사용하여 이진 트리의 오차를 보완하는 방식

* 깊이가 얕으므로 과대적합에 매우 강함

* 경사 하강법 사용, 분류: 로지스틱 손실 함수 사용, 회귀: 평균 제곱 오차 함수 사용

:::info 깊이가 얕은 트리를 사용하는 이유
경사 하강법에서, 그레이디언트 부스팅은 결정 트리를 계속 추가하며 가장 낮은 곳을 찾아 내려감. 이 때, 천천히 조금씩 이동해야 하기 때문에 깊이가 얕은 트리를 사용.(학습률 매개변수로 속도 조절 가능)
:::

## 히스토그램 기반 그레이디언트 부스팅

* 가장 인기 많은 알고리즘

* 입력 특성을 256개의 구간으로 나누기 -> 노드 분할 시 최적의 분할을 매우 빠르게 찾을 수 있음

* 이 중 한 구간을 누락된 값을 위해 사용 -> 누락 특성에 대한 전처리 필요 없음

* 트리의 개수 매개변수가 n_estimators가 아닌 max_iter 사용

## 앙상블 학습을 통한 성능 향상

## 우진

### 트리의 앙상블

> 랜덤포레스트
 - 앙상블 학습의 대표적인 알고리즘 중 하나로 안정적인 성능이 장점임
 - 결정 트리를 랜덤하게 만들어 결정 트리의 **숲**을 만들고, 각 결정 트리의 예측을 사용해 최종 예측을 만듬.
 - 랜덤하게 선택한 샘플과 특성을 사용 > 훈련 세트에 과대적합 예방, 검증 세트와 테스트 세트에서 안정적인 성능을 얻을 수 있음.


```python
import numpy as np
import pandas as pd
from sklearn.model_selection import train_test_split

wine = pd.read_csv('https://bit.ly/wine_csv_data')

data = wine[['alcohol', 'sugar', 'pH']].to_numpy()
target = wine['class'].to_numpy()

train_input, test_input, train_target, test_target = train_test_split(data, target, test_size=0.2, random_state=42)     # 세트 나누기
```


```python
from sklearn.model_selection import cross_validate
from sklearn.ensemble import RandomForestClassifier

rf = RandomForestClassifier(n_jobs=-1, random_state=42)

# return_train_score=True 지정 시 훈련 세트에 대한 점수도 같이 반환
# n_jobs=-1로 지정해두는 이유? : 기본적으로 100개의 결정 트리를 이용하기 때문
scores = cross_validate(rf, train_input, train_target, return_train_score=True, n_jobs=-1)      

print(np.mean(scores['train_score']), np.mean(scores['test_score']))
```

    > 0.9973541965122431 0.8905151032797809
    


```python
rf.fit(train_input, train_target)
print(rf.feature_importances_)      # 특성 중요도 출력
```

    > [0.23167441 0.50039841 0.26792718]
    


```python
# OOB (=Out Of Bag) : 부트스트랩 샘플에 포함되지 않고 남는 샘플. 이를 사용하여 부트스트램 샘플로 훈련한 결정 트리 평가 가능 (=검증 세트의 역할)

rf = RandomForestClassifier(oob_score=True, n_jobs=-1, random_state=42)     # oob점수를 얻기 위해선 true로 지정을 해줘야함.

rf.fit(train_input, train_target)
print(rf.oob_score_)
```

    > 0.8934000384837406
    

### 엑스트라트리
 - RF와 ET의 공통점 : 결정 트리가 제공하는 대부분의 매개변수를 지원하고 전체 특성 중에 일부 특성을 랜덤하게 선택하여 노드를 분할하는 데 사용함.
  
 - 차이점 : ET는 부트스트랩 샘플을 사용하지 않음 > 즉각 결정 트리를 만들 때 전체 훈련 세트를 사용하며, 노드를 분할할 때 가장 좋은 분할을 찾는 것이 아닌 무작위로 분할함.

 - ET의 특징 : 무작위성이 RF보다 더 크기 때문에 더 많은 결정 트리를 훈련해야 함. 그러나 랜덤하게 노드를 분할하기 때문에 빠른 계산 속도를 가질 수 있음.

> 하나의 결정 트리에서 특성을 무작위로 분할한다면 성능이 낮아지겠지만 많은 트리를 앙상블하기 때문에 과대적합을 막고 검증 세트의 점수를 높이는 효과를 가지고 있음.


```python
from sklearn.ensemble import ExtraTreesClassifier

et = ExtraTreesClassifier(n_jobs=-1, random_state=42)
scores = cross_validate(et, train_input, train_target, return_train_score=True, n_jobs=-1)

print(np.mean(scores['train_score']), np.mean(scores['test_score']))
```

    > 0.9974503966084433 0.8887848893166506
    


```python
et.fit(train_input, train_target)
print(et.feature_importances_)
```

    > [0.20183568 0.52242907 0.27573525]

 - 깊이가 얕은 결정 트리를 사용하기 때문에 **과대적합에 강하고 일반적으로 높은 일반화 성능을 기대할 수 있음**

> 정리
 - **앙상블 학습** : 더 좋은 예측 결과를 만들기 위해 여러 개의 모델을 훈련하는 ML 알고리즘

 - **랜덤 포레스트** : 대표적인 결정 트리 기반의 앙상블 학습 방법. 부트스트랩 샘플을 사용하고 랜덤하게 일부 특성을 선택하여 트리를 만드는 것이 특징.

 - **엑스트라 트리** : 랜덤 포레스트와 비슷하게 결정 트리를 사용하여 앙상블 모델을 만들지만 부트스트랩 샘플을 사용하지 않음. 대신 랜덤하게 노드를 분할해 과대적합을 감소시킴.
  
 - **그레이디언트 부스팅** : 랜덤 포레스트나 엑스트라 트리와 달리 결정 트리를 연속적으로 추가하여 손실 함수를 최소화하는 앙상블 방법. 이런 이유로 훈련 속도가 조금 느리지만 더 좋은 성능을 기대할 수 있음.

 - **히스토그램 기반 그레이디언트 부스팅** : 그레이디언트 부스팅의 속도를 개선한 것. 안정적인 결과와 높은 성능으로 인기가 높음.

* 트리 훈련에 사용될 훈련 세트의 비율을 정하는 subsample매개변수: default는 1.0, 기본적으로 전체 훈련 세트를 사용하지만 경사 하강법 단계처럼 일부 샘플을 랜덤하게 선택할 수 있음(확률적 경사 하강법, 미니배치 경사 하강법과 비슷)

* 일반적인 성능은 랜덤 포레스트보다 조금 더 높음. 단, 순서대로 트리를 추가해야 하기 때문에 훈련 속도가 느림(병렬작업 불가)

permutation_importance 함수 사용
```python
from sklearn.inspection import permutation_importance

hgb.fit(train_input, train_target)
result=permutation_importance(hgb, train_input, train_target, n_repeats=10, random_state=42, n_jobs=-1)

print(result.importances_mean)
```

    [0.08876275 0.23438522 0.08027708]
    
:::info permutation_importance 함수란?
* 특성을 하나씩 랜덤하게 섞어 모델의 성능 변화를 바탕으로 어떤 특성이 중요한지 계산

* 사이킷런의 모든 추정기 모델에 사용 가능

* n_repeats: 랜덤하게 섞을 횟수 지정(default=5)

* 반환 객체는 importances(중요도), 중요도의 평균, 표준 편차(importances_std)등을 갖는다

* 이러한 정보로 어떤 특성에 관심을 가질지 예상 가능
:::
